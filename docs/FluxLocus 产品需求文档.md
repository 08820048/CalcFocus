# FluxLocus 产品需求文档



---

**FluxLocus 产品需求文档（PRD）**  
**版本**：v0.3  
**日期**：2026 年 4 月  
**产品名称**：**FluxLocus**（流迹 / FluxLocus Studio）  
**Slogan**：流动的轨迹，数学级丝滑的专业录屏  
**定位**：一款高性能、跨平台、All-in-One 的屏幕录制与编辑工具，专注于自动光标轨迹美化（Locus）与动态流动特效（Flux），让演示、教程、产品 Demo 几分钟内即可输出电影级专业视频。

### 1. 产品背景与目标
- **灵感来源**：  
  主要参考以下优秀开源项目：  
  - **Recordly**（https://github.com/webadderall/Recordly）：跨平台屏幕录制 + PixiJS 驱动的光标特效、自动缩放、时间线编辑、运动模糊等成熟实现。  
  - **OpenScreen**（https://github.com/siddharthvaddem/openscreen）：专业级的自动/手动 zoom、注释、美化背景、一键专业导出等功能，以及整体产品 Demo 体验。  
  - Screen Studio 的核心用户体验与卖点（自动光标美化、一键电影级输出）。  

  在此基础上，结合 **数学美学**（样条曲线轨迹、精确插值动画、流动效果），使用 **Tauri + Rust** 重构，实现更轻量、高性能、丝滑的 All-in-One 体验。
- **核心价值**：极致轻薄、丝滑、高级质感的一体化操作，用户几乎无需切换页面，即可完成录制 → 特效调整 → 编辑 → 导出全流程。
- **目标用户**：
  - 开发者与产品经理（产品 Demo、功能演示）
  - 技术教程创作者、教师
  - 短视频内容制作者（B站、TikTok、Twitter 等）
- **设计理念**：**All-in-One 单页面** + **轻薄丝滑高级质感**（Lightweight + Smooth + Premium），默认深色模式，融入 Liquid Glassmorphism（液体玻璃）风格与数学几何元素。

### 2. 技术栈
- **整体框架**：**Tauri 2**
- **后端**：**Rust**  
  （scap / xcap 屏幕捕获、ffmpeg-next 视频编码、nalgebra / glam 数学计算、tokio 异步）
- **前端**：**React 19 + TypeScript + Vite**  
  - UI：Tailwind CSS 4 + shadcn/ui + Radix UI  
  - 状态管理：Zustand  
  - 动画：Framer Motion  
  - 实时特效预览与渲染：**PixiJS v8**（参考 Recordly 的 PixiJS 实现，并借鉴 OpenScreen 的 zoom 与美化逻辑）
  - 路由：TanStack Router（可选）
- **通信**：Tauri invoke / event（Rust 与前端交互）
- **打包**：Tauri 官方构建（目标 Bundle 体积远小于 Electron）

### 3. UI/UX 设计风格
**设计原则**：
- **All-in-One 单主界面**：所有功能集成在一个主要窗口内，最大程度减少页面切换。
- **轻薄（Lightweight）**：极简布局、大留白、细线条、无厚重卡片。
- **丝滑（Smooth）**：所有交互使用高帧率缓动动画（Framer Motion + PixiJS GPU 加速）。
- **高级质感（Premium）**：以 **Liquid Glassmorphism**（液体玻璃效果）为主，半透明毛玻璃面板 + 强背景模糊 + 细腻辉光边框。
- **数学风注入**：干净几何线条、微弱样条曲线装饰、淡雅 Flux 流动渐变（仅在预览或 hover 时轻微出现）。

**颜色系统**：
- 主背景：极深黑（#050505）
- 面板：半透明深灰（rgba(20,20,30,0.85)）+ backdrop-blur-2xl
- 强调色：Flux 电光蓝紫渐变（#60A5FA → #A78BFA），Locus 青绿（#34D399）
- 文字：#F1F5F9（主） / #94A3B8（次要）

**整体布局结构**（Single Page）：
- **顶部栏**（极薄 48-56px）：Logo + 项目名称（可编辑）+ 录制控制按钮（Record / Pause / Stop）+ 设置 + 导出 CTA
- **左侧面板**（240-280px，可折叠）：录制设置 + 特效快捷开关（Locus / Flux）+ 摄像头控制
- **中央主区域**（占比最大）：PixiJS 实时/编辑预览画布 + 浮动播放控制条
- **底部时间线**（180-240px，可拖拽展开）：波形轨道 + 裁剪 + 关键帧 + 轨迹可视化
- **右侧属性面板**（260px，可折叠）：上下文敏感的样式与数学参数调整（轨迹平滑度、模糊强度等）
- **专注模式**：一键隐藏侧边栏，仅保留中央预览 + 底部时间线 + 浮动工具栏

**交互原则**：
- 所有面板展开/收起、拖拽、参数调整均使用丝滑动画（300-400ms ease-out）。
- 按钮 hover 带轻微 scale + 辉光反馈。
- 时间线支持磁吸对齐与实时预览更新。
- 支持全局快捷键 + 命令面板（Cmd/Ctrl + K）。

### 4. 核心功能需求

#### 4.1 录制模块
- 支持全屏、单个窗口、自定义区域录制
- 同时录制系统音频 + 麦克风 + 摄像头（可选）
- 平台原生捕获：macOS ScreenCaptureKit、Windows WGC、Linux PipeWire
- 实时控制：开始/暂停/停止、倒计时、快捷键
- 录制完成后自动进入编辑模式，并保存源媒体

#### 4.2 光标与轨迹特效（Locus 核心）
- 平滑光标叠加（大小、样式可调）
- **数学级轨迹平滑**：样条曲线（Spline / Catmull-Rom / Bezier）处理光标运动轨迹（升级 Recordly 的轨迹逻辑）
- 运动模糊、点击高亮、自动隐藏静止光标
- **Locus 可视化模式**：可选在预览中绘制光标运动的数学轨迹曲线（教学风）

#### 4.3 自动聚焦与流动效果（Flux 核心）
- **智能自动缩放**：根据鼠标活动（速度、点击等）自动检测重点区域并平滑 zoom（借鉴 OpenScreen 的 zoom 机制，并升级为数学插值）
- Flux 流动特效：动态模糊、渐变过渡、粒子/流动高亮（PixiJS 预览）
- 自动生成缩放建议（时间线上显示为数学标记点）

#### 4.4 时间线编辑器
- 拖拽式时间线（参考 Recordly 的 PixiJS 时间线实现）
- 基础操作：裁剪片段、调整播放速度（变速区域）、画面整体裁切
- 添加手动缩放关键帧、文本/形状/图片注释（借鉴 OpenScreen 的注释系统）
- 额外音频轨道叠加
- 项目文件（.fluxlocus）保存所有编辑状态，支持断点续编

#### 4.5 摄像头叠加
- 气泡式叠加，支持位置、大小、圆角、阴影、镜像调整
- 支持与画面缩放联动的动态效果（参考 OpenScreen 与 Recordly）

#### 4.6 画面样式与美化
- 背景：纯色、渐变、自定义图片、模糊效果（借鉴 OpenScreen 的美化能力）
- 画面留白、圆角边框、投影阴影
- 宽高比预设（16:9、9:16、1:1 等）
- Flux 风格数学渐变背景（可选）

#### 4.7 导出与分享
- 支持导出 MP4（最高 4K、可调码率）和 GIF
- 可调分辨率、质量、宽高比
- 导出进度展示 + 自动打开输出目录

#### 4.8 辅助功能
- 全局快捷键与命令面板
- 设置面板（主题、性能、语言、中英双语）
- 噪声抑制（Rust 后端处理）
- 更新检查与自动更新

### 5. 非功能需求
- **性能**：录制与预览时低 CPU/GPU 占用，支持 4K/60fps；Bundle 体积目标 < 50MB（Tauri + Rust 优势）
- **兼容性**：macOS 12.3+、Windows 10 20H1+、主流 Linux（PipeWire）
- **隐私与安全**：全部本地处理，清晰的权限申请流程
- **可访问性**：键盘友好、高对比模式支持
- **开源与许可**：推荐 MIT / Apache 许可。开发中参考 Recordly（AGPL-3.0）和 OpenScreen 的实现思路，但注意许可证兼容性，避免直接复制代码。如大量借鉴，建议与原作者沟通或保持核心功能独立实现。

### 6. 开发阶段规划（MVP 路线）
**MVP（Phase 1）**：
- All-in-One 主界面框架（玻璃质感 + 布局）
- 基础录制 + 简单编辑 + 导出
- 基础光标渲染 + 样条轨迹平滑（参考 Recordly PixiJS）
- PixiJS 实时预览

**Phase 2**：
- Flux 流动特效 + 智能自动缩放（借鉴 OpenScreen zoom）
- 完整时间线与属性面板
- 项目文件保存 + 样式系统

**Phase 3**：
- Locus 轨迹可视化教学模式 + 高级数学动画
- AI 增强功能（可选：自动字幕、智能剪辑）
- 多平台深度优化与社区贡献
